# LESSON-014 · 错题瞬间 = 3 件事原则

## 🔴 NZH 设计哲学(2026-04-26 立法)

> **学生答错时只需要 3 件事**:
> 1. **看答案**(主动点 · 不强弹)
> 2. **知道为什么错**(简洁 · 1 句)
> 3. **再来一题**

> **所有其他都是噪音**:
> - 错因分析 = **系统后台记** · 不在用户面 surface
> - 错题推荐 / 类似题 = 整个 session **完成后** 才推荐 · 不每错弹 · 那很烦
> - step-by-step 教学 = 学生**主动点"查看"**才出现 · 不默认占屏
> - 复习推荐 / 类似真题 = 同上 · 留给 session 结束 / KP 详情页

## 灵魂红线分析

NZH 截图触发的 7 块过载:

| 块 | 性质 | 灵魂红线触犯 |
|---|---|---|
| 卡住的根本原因卡(整个紫卡 + 4+3 标签) | 错题推荐 | 红线 1 焦虑特性 |
| ErrorDiagnosticsCard | 错因卡片 | 红线 1 + 红线 5 "你应该早就会了" 隐喻 |
| WhyStuckPanel | 为什么卡住 | 同上 · "为什么"问 = 评价学生 |
| ConceptTag(每题显)| 该 KP 标识 | 红线 7 variant 1=1=1 反向 · 噪音 |
| ✨ 跟我看一遍 CollapsibleSection | step-by-step inline | 默认占屏 = 学生没主动求助但被强 surfacing |
| 📋 真题回顾 CollapsibleSection | 历史错题 | 错位:答错瞬间不是回顾历史时机 |
| 浮按钮 "我有点卡住了"(我加的)| belief 触发 | 又一个 + 1 噪音 · NZH 直觉对 |

## 规则

**Why**:错题瞬间是**情感瞬间** · 不是**教学瞬间**。学生答错时 · 大脑 cortisol ↑ · 任何额外信息都是噪音。**老师视角**:好老师不会拿 5 张卡片砸刚答错的学生 · 一句"再来一次"+ 留路径(学生想看再点)。

**How to apply**:

错题瞬间 UI 必过 3 问:

1. **学生现在能不能 1 秒内决定下一步?**(再来 / 看答案 / 跳过)
2. **这个组件是 push 还是 pull?**(主动 surface = push = 噪音 · 学生点才出 = pull = OK)
3. **这个信息能不能挪到 session 总结 / KP 详情页 / 后台衍生层?**(能挪 = 必挪)

## 错题瞬间 UI 模板(灵魂宪章合规)

### 必有 3 件
1. **简洁反馈** · "还不对"(不教训语)+ 1 句温度("没关系" / "再来")
2. **学生主动入口**:
   - 文字链 "查看答案"(轻 · 仅亮答案)
   - 按钮 "看例题讲解"(重 · 弹 modal · 含答案 + step-by-step)
3. **主 CTA** "下一题"(大蓝 / 蓝紫色)

### 不准有
- ❌ 错因诊断卡(后台记 · 不 surface)
- ❌ 类似题推荐(留 session 总结)
- ❌ 复习路径推荐(留 session 总结)
- ❌ 错题历史回顾(留专门页)
- ❌ KP 标识 chip 每题显(噪音)
- ❌ CollapsibleSection 默认 inline 占屏
- ❌ 浮按钮 / 弹窗 / toast 主动 surface 教学内容

### 可选(学生主动 trigger)
- ✅ 学生主动点"看例题讲解" → modal 详细
- ✅ 学生主动点"我有点卡住了" → belief 三选(但应放 session 总结而非每错出现)

## 🔴 克制原则(NZH 2026-04-26 二度升级)

> **太热情 = 太过载 · 要克制。**
> **不需在每道题显示历年真题怎么考。**
> **不需全列所有历年真题。**
> **入口要整合 · 不要 A "看答案" + B "看解析" 双入口 · 学生不需选 2 次。**

### 克制 3 准则

1. **不每题塞所有信息**:历年真题 / 类似题 / 复习路径 · 都不在每题瞬间塞 · 学生不一定需要
2. **入口去重整合**:看答案 + 看解析 = **1 个入口**(打开后既见答案又见解析)
3. **少即是多**:能折叠的不展开 · 能撤的撤 · 能挪的挪 · 默认极简

### 单一答案+解析入口 UI 模板

```
[ 👁 看答案与解析 ]  · 学生点击
       ↓
   modal 弹出 · 顶部高亮"答案: X"
       ↓
   下方 step-by-step 解析(Walkthrough)
```

不要:
- ❌ "查看正确答案" 文字链
- ❌ "看例题讲解" 按钮(同时存)
- ❌ 之后再加"看类似题的讲解"(更多)

要:
- ✅ 1 个按钮 "看答案与解析"
- ✅ modal 内整合呈现

## 错题反馈与 session 总结的分工

| 时机 | UI 责任 |
|---|---|
| **错题瞬间** | 看答案 / 知道为什么错(简洁)/ 再来一题 |
| **session 总结** | 错题归类 + 推荐复习 + belief 三选 + 鼓励 |
| **KP 详情页** | step-by-step 详细教学 + 类似真题 + 历史 |
| **后台衍生层** | 错因 pattern 累积 / 推荐计算(NZH 看 dashboard 时 surface) |

错位 surface = 时机错 = 噪音。
