# LESSON-015 · 整站不过载 5 铁律

## 🔴 NZH 立法(2026-04-26 · LESSON-014 整站升级)

LESSON-014 立法"错题瞬间 = 3 件事"· 但学生在整站(Dashboard / 课程 / 作业 / 答题 / 总结)每个情境都可能过载。本 LESSON 把 014 扩展为**整站铁律**。

> **核心公式**:**后端做满 · 前端做减 · 语气做柔 · 节奏做缓**
> · 学生进 Practice 不是去"看数据" · 是去"被陪伴"

---

## 铁律 1 · 情境-即-界面(Context-Is-UI)

> 学生在每个时刻只能处于一种情绪状态 · UI 只服务这个情绪 · 其他全部下沉

### 6 情境矩阵

| 时刻 | 情绪 | UI 首屏 | 不准出现 |
|---|---|---|---|
| Dashboard 进入 | 找路 | 3 类聚合卡(今日/课程/作业)+ 详情下钻 | 排名 / 百分比 / 数据看板 |
| 选题前 | 决定 | 1 主 CTA + ≤ 2 辅助(低对比度) | 多档同等权重 / 全列真题 |
| 答题中 | 专注 | 1 题 + 1 输入 + 1 提交 | 进度数字 / 用时 / 同班对比 |
| 答错瞬间 | 受挫 | 看答案&解析 / 再来一题(LESSON-014) | 错因 chip / 推荐 / 真题列表 |
| 答对瞬间 | 庆祝 | 1 句温度 + 下一题 | "你已超过 X% 同学" |
| 总结时 | 反思 | 学过 / 错过 / 修过(三类计数 · 不显 %) | 实时焦虑指标 / mastery score |

### 原则

- **首屏 ≤ 3 类聚合** · 不是硬元素上限 · 详情下钻
- **滚动后才能出衍生内容** · 不在首屏堆叠
- **每情境 1 主 CTA** · 多 CTA = 噪音

## 铁律 2 · 分角色铁律(Role-Gated Surface)

> 后端数据在三角色(学生 / 教师 / 家长)前 surface 度不同 · 不互通

| 角色 | 可见数据 | 不可见 |
|---|---|---|
| **学生** | 学过 / 错过 / 修过(计数)+ 当前推荐(1 句话)+ 自己当前答题 | mastery_score / confidence / belief / error_pattern / 同班对比 |
| **教师** | 班级热力图 + 个人 dashboard 全维 + error_pattern + 修复路径 | 家长私聊数据 |
| **家长** | 孩子学过 / 修过(计数)+ 趋势(向上)+ NZH 留言 | 弱项 / 排名 / 实时 mastery 数字 |

### 原则

- **后端跑满 8 维评估**(KP/FLM/mastery/belief/next-step/error-pattern/vocab-pool/scaffold-tier)
- **学生侧只 surface "1 句话 + 1 按钮"**
- **教师侧 surface 全量**(老师角色不焦虑数据)
- **家长侧仅 surface 正向**(学过/修过 · 不 surface 弱项防焦虑)

### 反例(2026-04-26 之前)

```
学生 Dashboard 显示:
- "你的 KP-0142 mastery 是 42%"  ← 学生看不懂 + 焦虑
- "你比同班 73% 同学慢"          ← 灵魂红线 4 触犯
- "建议练习强度 +30%"            ← 灵魂红线 1 焦虑特性
```

### 正例(本 LESSON 立法后)

```
学生 Dashboard 显示:
- "今天来 5 分钟,我们继续 Y10.3?"  ← 1 句话 + 1 按钮
(后端 8 维全跑 · 但只 surface 推荐结果)
```

## 铁律 3 · push 白名单(Pull-First With Narrow Push Whitelist)

> 默认所有内容 **pull(学生主动)** · 仅以下 3 类允许 **push(主动 surface)**

### push 白名单(仅 3 类)

1. **时间敏感作业截止** · 例:"明天 23:59 前 · 课后作业 Y10.3"
2. **教师课堂实时推送** · 例:"NZH 现在让全班练这 5 题"
3. **学生上次未完成** · 例:"上次到 Y10.3 第 2 题 · 继续?"

### 永远 pull 的内容

- ❌ 教学内容(walkthrough / tutorial / 例题)— 学生主动点
- ❌ 错因分析 / 推荐路径 — 学生主动求助
- ❌ 真题列表 / 历史回顾 — 学生主动展开
- ❌ 词汇查词 — 学生主动点词
- ❌ 衍生层数据(error_pattern / mastery 趋势)— 教师 dashboard 才 surface

### 原则

- **push 必须时间敏感或教师授权** · 否则一律 pull
- **toast / modal / 弹窗** 仅用于 push 白名单 · 教学内容禁用
- **红点 / streak / 倒计时** 仅作业截止可用 · 学习内容禁用(灵魂红线 2)

## 铁律 4 · 纯继续语(Pure Continuation Voice)· 🔴 绝对红线

> **NZH 2026-04-26 二度强化**:
> **绝对去掉"你已多少天未练习"这样的语言 · 任何时候都欢迎回来 · 继续推荐"练习哪个专题"**
>
> 学生离开-回来 · 系统永远用 **"继续语"** 不用 **"提醒语"** · 不暴露时间间隔

### 绝对禁止清单(任一出现 → 阻 PR · 紧急修)

- ❌ "你已 X 天未练习"
- ❌ "你已 X 周未登录"
- ❌ "好久不见"
- ❌ "你最后一次练习是 X 天前"
- ❌ "Streak 已断" / "断签"
- ❌ "今日打卡" / "连续打卡"
- ❌ "Daily streak" / "Login streak"
- ❌ "完成今日目标" 类 daily/weekly 强制节奏语
- ❌ 任何包含具体 时间间隔数字 + "未/没/没有" 的组合
- ❌ "回来" 前加任何时间修饰("终于"/"久违"/"过了这么久")
- ❌ "提醒你"(含暗示遗忘的语义)

### 必用替换(任何"回来"场景)

| 场景 | 必用文案 |
|---|---|
| 学生重新登录(无论间隔)| "嗨 · 继续 Y10.3 第 2 题 吗?" 或 "今天我们做 [推荐专题]?" |
| 上次未完成 session | "我们上次到 [unit] 第 N 题 · 继续吗?" |
| 推荐专题(有 next_step)| "今天推荐你练 [专题名]" |
| 推荐专题(无 next_step) | "今天我们做哪个?" + 3 个专题入口 |
| 完全新用户 | "嗨 · 从哪开始?" + 入口卡 |

### 灵魂原则

- **任何时候都"欢迎回来"** · 不带条件 · 不暴露时间
- **永远 forward-looking**:下一步是什么 / 推荐哪个专题
- **绝不 backward-looking**:不提"过去多久" / "上次什么时候"(除"我们上次到第 N 题"这种纯进度引用 · 不含时间)
- **不评价节奏**:不出现 "快/慢/落后/领先/应该/本来"
- **不 push 时间焦虑**:截止日期可显(白名单)· 但学习节奏不可显

### 反例 → 正例转写

```
❌ "好久不见!你已 12 天未练习,继续保持学习习惯吧"
✅ "嗨 · 今天我们继续 Y10.3 吗?"

❌ "你的连续打卡已断 · 今天再来一次?"
✅ "今天推荐你练分数除法 · 5 分钟搞定"

❌ "上次练习是 2026-04-14 · 距今 12 天"
✅ "上次我们停在 Y10.3 第 5 题 · 继续吗?"

❌ "完成今日目标:30 分钟"
✅ (撤 · 不显日目标)
```

### 实施位置(代码层强制)

- `src/lib/soulMessages.ts` · welcomeBack / continueSession 文案池 · 必过本 LESSON 检查清单
- `src/pages/Dashboard.tsx` · 顶部欢迎语 · 必用"嗨 · 继续 X 吗?" 模板
- `src/pages/Login.tsx` 登录后跳转 · 不显时间间隔
- 任何 toast / banner / modal · grep `天未` / `streak` / `好久` / `daily` 必为 0

### 灵魂红线对应

- **触犯灵魂红线 1**(焦虑特性)
- **触犯灵魂红线 2**(红点 / streak / 倒计时反向驱动 UI)
- **触犯灵魂红线 5**("你应该早就会了" 的时间版本:"你应该早就来了")

### 替换表

| 提醒语(禁)| 继续语(用)| 为什么 |
|---|---|---|
| "你已 7 天未练习" | "我们上次到 Y10.3 第 2 题 · 继续吗?" | 不暴露时间 = 不内疚 |
| "你落后于学习计划" | "今天我们做哪个?" | 不评价 |
| "好久不见!" | "嗨,继续吗?" | 不强调"久" |
| "你最近表现下降" | (撤 · 不 surface)| 灵魂红线 5 |
| "complete your daily streak" | (撤 streak 概念)| 灵魂红线 2 |

### 原则

- **不暴露时间间隔**(7 天 / 1 周 / 上周)
- **不评价节奏**("快" / "慢" / "落后" / "领先")
- **不引内疚**("应该" / "本来" / "差一点就")
- **永远 forward-looking**(下一步是什么)· 不 backward-looking(过去多久没来)

## 铁律 5 · 双层数据(Dual-Layer Data Architecture)

> 后端闭环必须有 "学生不可见但教师可见" 的双层数据架构 · 不是 backend 全显或全藏

### 数据分层

| 层 | 内容 | 学生 | 教师 | 家长 |
|---|---|---|---|---|
| **L1 学生面层** | 学过 / 错过 / 修过(计数)+ 当前推荐 | ✅ | ✅ | ✅ |
| **L2 教师诊断层** | mastery_score / error_pattern / confidence / FLM state | ❌ | ✅ | ❌ |
| **L3 算法过程层** | belief / next-step 决策树 / scaffold tier 计算 | ❌ | ✅(可折叠)| ❌ |
| **L4 家长汇报层** | 学过/修过趋势 + NZH 留言 | ❌ | ✅(教师写)| ✅ |

### 原则

- **后端 8 维评估全跑** · 不为简化前端而砍后端
- **L1 是"陪伴语"** · L2/L3 是"诊断数据" · L4 是"成长汇报"
- **不在学生面 surface L2/L3** · 即使学生主动问 · 也用 L1 措辞回答
- **教师 dashboard 可下钻 L2/L3** · 但默认折叠 L3(教师不需算法细节)

### 反例

```
学生主动问:"我哪里没掌握?"
错答:"你的 KP-0142 mastery 是 42% · belief 不稳"  ← 暴露 L2/L3
正答:"我们一起再做 3 题 Y10.3 第 5 题型 · 看看?"  ← L1 陪伴语
```

## 灵魂自检(每个新页面 / 每个 PR)

- [ ] **情境对吗**?学生此刻在哪个情绪 · UI 服务这个情绪吗?
- [ ] **角色对吗**?这是学生侧还是教师侧 · 数据 surface 度对吗?
- [ ] **push 合规吗**?这个主动 surface 在白名单内吗?(否则改 pull)
- [ ] **语气对吗**?有"继续语"还是"提醒语"?暴露时间间隔了吗?
- [ ] **L1 还是 L2**?学生看到的是陪伴语还是诊断数据?
- [ ] **首屏 ≤ 3 类**?详情下钻而非堆叠?

任一未过 → 重审。
