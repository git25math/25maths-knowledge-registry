# LESSON-016 · 哈罗课纲学习线 8 模块内容标准

## 上下文

NZH 经 6 轮 UI 打磨定稿 v6 阶段化 UnitDetail · 8 模块组织如下:

```
课前   ●  词     ●  知识点
课中   ●  例题   ▶ ●  手把手
课后   ○  配套   ○  专题   ○  真题
拓展   ○  延伸阅读
```

本 LESSON 立每模块的**内容标准 + UI 规格 + 反例**,作为 NZH 团队产出依据 · 跨 session 复用。

---

## 共性铁律(模块通用)

- 模块页 chrome:`Y10.3 · 分数除法 · {子位置}` + 底 `→ 下一节名`
- **0 副标 / 0 emoji 头 / 0 元数据 surface / 0 时长估算**
- 双语:中英**独立写 · 不互译**
- 完成态:**静默判定**(后端规则)· 学生只见 `●○▶✓`
- 反馈语:全站统一 5 档(`feedbackHeaders` · LESSON-014 配套)
- 学生主体:无"老师推 / 系统建议 / 我们为你"等系统旁白

## 🔴 双层命名铁律(后端 L1/L2/L3 vs 前端基础/进阶/综合)

**后端 / 数据 / API**:`L1` `L2` `L3`(代码常量 · 永不变 · 跨产品)

**前端学生侧**:

| 后端 | 中文 | 英文 |
|---|---|---|
| L1 | 基础 | Basic |
| L2 | 进阶 | Standard |
| L3 | 综合 | Apply |

**教师 dashboard**:双标 `L1 · 基础` / `L2 · 进阶` / `L3 · 综合`(便于跨系统对比)

**禁用**(防焦虑 / 防混淆):
- ❌ `Foundation`(与 IGCSE Tier 撞)
- ❌ `Advanced` / `较难` / `Hard`(暗超纲焦虑 · LESSON-014 红线 5)
- ❌ `Easy` / `容易`(贴标签)
- ❌ 学生侧裸显 `L1/L2/L3`(后端编号 · 不直接 surface)

## KP × Level 架构(2026-04-26 v7 升级)

每 unit 含多 KP · 每 KP 含多 level · 每 level 闭环三步:

```
Unit
├─ 词 / 知识点(unit 级)
├─ KP × N(教学知识点 · 1-5 个)
│   └─ Level × M(题型难度 · 每 KP 1-3 个)
│       ├─ 例(walkthrough 看一遍)
│       ├─ 练(举一反三 · 答对 ≥ 3)
│       └─ 做(课后 5 题独立)
├─ 单元回顾(全 KP 综合 · 5 题混合)
├─ 错题专练(条件显 · ReviewQueue 推)
└─ 专题 / 真题 / 延伸(unit 级 · 跨域)
```

**显示 KP** vs **后端 kn_id**:
- 学生看 1-3 个 displayKP(粗粒度 · 教学单元)
- 后端跑 24 个 kn_id mastery(细粒度 · 数据精度)
- LESSON-015 双层数据自然形成

**信息深度梯度**:
```
词 → 知识点 → KP1.L1.例 → 练 → 做 → KP1.L2 → ... → 回顾 → 错题 → 专题 → 真题
浏览          阅读        观察    跟做  独做                  综合     针对     跨域   挑战
```

## 模块 1 · 词

| 项 | 标准 |
|---|---|
| 数量 | 6-10 词 / unit · 超 10 拆 sub-unit |
| 来源 | CIE 228 套卷高频词 + NZH 课纲补关键词 |
| 每词必有 | 英 + 中 + NZH 写的 1 句用法 |
| L2 可有 | 真题句式 ≥ 1(从真题原句切)|
| L3 可有 | 跳关联 KP 的"知识点"模块 |
| 一词多义 | 仅显数学语境义 |
| UI | 列表式 · 行高 48px · 整行可点弹三层 popup |
| done 判定 | 进入 + 离开(不强测)|

**反例**:词典风长定义 / 中英堆叠 / 计时模式 / 进度条 / 多义堆叠

## 模块 2 · 知识点

| 项 | 标准 |
|---|---|
| 写作 | NZH 口语 · 短段 · 不书面 · 不教科书 |
| 结构 | 核心概念 1 句 → 公式 1-2 行 → 1-3 例 → (可选)反思 |
| 长度 | 1-2 屏 · 移动端 ≤ 3 屏 |
| 段落 | ≤ 6 段 |
| 公式独行 | ≤ 3 个(关键才独行)|
| 子标题 | ≤ 2 级 |
| 双语 | 中英独立写 · 不互译 |
| 创作 | NZH 写 MD · 脚本编 JSON |
| done 判定 | 滑底 + 停 5s |

**反例**:
- "💡 NZH 给你讲一遍" 类副标
- "本节学习目标:① ... ② ..."
- 教科书定义("Definition: For any non-zero rational number...")
- "技巧 / 口诀 / 小贴士"套路块
- "知识点回顾 / 本节小结"收尾段
- 每段 emoji 装饰

**反 / 正例**:
```
❌ Definition: The reciprocal of a non-zero rational number a/b
   is the number b/a, such that (a/b) × (b/a) = 1.

✅ 倒数就是把分子分母对调。3 的倒数是 1/3 · 它们相乘 = 1。
```

## 模块 3 · 例题

| 项 | 标准 |
|---|---|
| 题数 | **1 道**(LESSON-014 克制)|
| 步数 | 3-6 步 · 每步 1 个动作 |
| 题源 | CIE 真题典型题(中等档 · 不取顶级)|
| 步内容 | 动作名(短动词)+ 数学 · 不含解释段 |
| 学生交互 | 仅点"下一步"· 不答题 |
| 复用 | `StepByStepWalkthrough` 0 新代码 |
| done 判定 | 走完最后一步 |

**步骤动作命名**:
- ✅ "把混合数转成假分数" / "除变乘 · 取倒数" / "相乘" / "化简"
- ❌ "我们需要先把混合数转成假分数 · 因为这样才能 ..."(解释段)
- ❌ "Step 1: First, we ..."(教学腔)
- ❌ "为什么要先转换 ? 因为 ..."(模拟学生提问)

**反例**:>6 步 / 卡内多段解释 / 步间塞"易错点" / 走完弹"恭喜" / 多题堆叠 / 给学生选解法

|---|---|

| done 判定 | 答对 ≥ 3 题(任意题序)|
| UI | 题 + 输入 + 💡 + 提交 |

**反例**:错因 5 卡(LESSON-014 撤)/ ConceptTag / WhyStuckPanel / 真题历史回顾 / 浮按钮"我有点卡住了"

## 模块 5 · 配套

| 项 | 标准 |
|---|---|
| 题源 | V2 generators · `?scaffold=null`(无 walkthrough)|
| 题数 | **5 题**(不超 7 · LESSON-014)|
| 进度显 | 5 圆点 `● ◐ ○`(顶部)|
| 错题 | 自动入 ReviewQueue · 在"成长足迹" tab 看(LESSON-015 双层)|
| 中途退 | 静默存进度 · 0 弹窗 |
| done 判定 | 5 题做完(对错不限)|

**反例**:>7 题 / 显当前正确率 / 显已用时 / 显"已完成 X/5"冗余文字

## 模块 5b · 单元回顾(v7 新增)

| 项 | 标准 |
|---|---|
| 触发 | 该 unit 所有 KP × levels 全 ✓ 后 ▶ ● 切到此 |
| 题数 | **5 题混合** |
| 抽题策略 | 从已完成 levels 的 generator 池**随机抽 5**(默认平均分配 KP)|
| 不显 | 每题来自哪个 KP / level(防焦虑分类)|
| 错题 | 答错的入 ReviewQueue · 触发"错题专练"行(若新错)|
| done 判定 | 5 题做完(对错不限) |
| UI | 同模块 5 配套 · 5 圆点进度 |

**反例**:
- ❌ 显"本题来自 KP1.L2 倒数"
- ❌ 加权随机按 mastery 弱项偏置(暗压力)
- ❌ 答错弹"建议复习 X" toast(LESSON-014)

## 模块 5c · 错题专练(v7 新增)

| 项 | 标准 |
|---|---|
| 显条件 | **该 unit 在 ReviewQueue 累积 ≥ 1 错** · 否则整行从 UnitDetail 撤 |
| 触发位置 | UnitDetail "错题" 阶段一行 · 数量动态显("你的 N 道针对")|
| 题源 | `useUnitErrors(unitId)` 从 ReviewQueue 取该 unit 错题 |
| 题数 | 该 unit 当前所有 ReviewQueue 错题(N 题动态)· 不固定数 |
| 进度 | N 圆点(随题数变化)|
| done 判定 | 当前所有错题做完(对错不限) |
| 答完后 | 部分清 ReviewQueue · 错题继续在"成长足迹"留底 |
| 整体效果 | 学生 unit 内即清错 · 无须跳"成长足迹" |

**反例**:
- ❌ 0 错时显"暂无错题"占位(LESSON-014)
- ❌ 错题旁注错因 chip("方法错"/"概念错")(LESSON-015 双层 · 老师才看)
- ❌ 题号顺序按错率(变动焦虑)· 应固定按错的时序

## 模块 6 · 专题

| 项 | 标准 |
|---|---|
| 实现 | 跳现 `KnowledgeMap?focus={cieSection}` deeplink |
| 0 新页 | 模块本身只是入口 · 不在 HHK 内渲染 |
| 学生体验 | 学习线 → 备考线 跨域看 · 同 specRef 跨考纲题型 |
| 跳转后 | KnowledgeMap 顶部加返回 chip:`← 回到 Y10.3` |
| 返回路径 | 学生从 KnowledgeMap 点 chip = 回 UnitDetail · 该模块标 ● |
| 多 cieSection | 该 unit 关联多个 cieSection 时 · 跳 KnowledgeMap 用第一个 · 不让学生选(减决策) |
| done 判定 | 进入即 done(进入瞬间标 ●) |

**反例**:
- ❌ 在 HHK 内复制 KnowledgeMap 数据(冗余)
- ❌ 学生跳出后失去回路径(必须有"回 Y10.3" chip)
- ❌ 多 cieSection 时让学生选哪个(决策负担)
- ❌ 显"该专题在备考线还有 N 题"等 paper count 元数据

## 模块 7 · 真题

| 项 | 标准 |
|---|---|
| 题源 | `findVariantIdsForPaperQuestion` 反查 cieSections |
| 排序 | 按 EXAM_REFS `weight`(highest > high > medium > low > rare)|
| 显 | **top 3 默认** + "+ N 早年"折叠展开 |
| 每行 | `年期 · Qn  Xm  →`(整行可点)|
| 已答 | 该行变灰 · 不删 · 不显"已做"标(灰色即标识)|
| 答完返回 | 学生点真题 → PracticePage → 答完 → 自动回真题模块页(stack 弹回)|
| 多空(0 真题)| **整模块从 UnitDetail 撤** · 不显空架 |
| 单空(1-2 题)| 显 1-2 行 · 不强凑 3 |
| 折叠展开 | 点"+ N 早年"展开后 · 不再折叠(避免反复点)|
| 路径完结 CTA | 底部双 CTA `← Y10` / `Y10.4 →`(后者推未走过的下一 unit)|
| done 判定 | 答 ≥ 1 题(对错不限) |

**反例**:
- ❌ ★★★ 装饰评级
- ❌ "高频考点 / 偶有 / 难度高" 标签
- ❌ "对应 CIE Spec s1.4" 文字 surface
- ❌ 全列所有真题(>5)
- ❌ 显 marks 总量 / paper 来源链接 / 解析预告
- ❌ 答完弹"恭喜你答完真题"
- ❌ 用 emoji 区分新旧 / 难易

## 模块 8 · 延伸(可选)

| 项 | 标准 |
|---|---|
| 显条件 | 仅 NZH 在 section.json 填了 `extension` 字段才显 · 否则**整模块从 UnitDetail 撤** |
| 内容类型 | (a) 内部跳:跨 unit / 跨 KnowledgeMap deeplink<br>(b) 外部链接:视频(B 站/YouTube)/ 阅读(博客/文档)<br>(c) 嵌入:NZH 写的拓展 MD(同知识点格式)|
| 数量 | ≤ 3 个条目(LESSON-014 克制)|
| 每条目 | 标题(双语) + 1 行说明 + 类型图标 |
| 类型图标 | `📖` 阅读 / `🎬` 视频 / `→` 内部跳(撤其他)|
| 外链审核 | NZH 加的链 · 每条标 `verified: true/false` · 学生侧仅显 verified |
| 学生主动性 | 学生点才打开外链 · 不 autoplay 视频 · 不预加载 |
| MVP | 不强求 · 留 schema 架子 |
| done 判定 | 进入模块页即 done(无论是否点条目) |

**反例**:
- ❌ 显"暂无延伸内容"占位(LESSON-014)
- ❌ NZH 没审就放外链
- ❌ video autoplay
- ❌ "推荐你看"等推送语(LESSON-015 pull 优先)
- ❌ 显观看时长 / 阅读时长(时间焦虑)
